# 智能体意图分解+工具选择模块设计文档（TS版）

## 文档说明
本文档包含智能体「意图分解子智能体」和「工具选择子智能体」的完整设计、核心实现代码及对接逻辑，适配基于TypeScript的LangChain/Agent Skill工具体系，严格遵循「抽象与实现分离、模块解耦」原则。

---

## 一、核心设计原则
1. **解耦优先**：意图分解仅依赖「工具抽象能力」，不接触具体工具实现；工具选择负责匹配具体工具，两者仅通过结构化数据对接。
2. **渐进式披露**：工具选择时严格遵循「基础层优先、匹配度次之」的策略，降低智能体调用风险。
3. **结构化输出**：所有模块输出标准化JSON结构，确保对接稳定性。
4. **上下文兼容**：全流程支持历史会话传入，保证意图理解和步骤拆分的准确性。

```typescript
/**
 * 工具注册表
 * 负责工具注册、抽象元数据暴露
 * 是意图分解与工具体系的唯一对接点
 */
import { BaseToolMeta, ToolCapabilityMeta, ToolLayer } from "../../common/types";

export class ToolRegistry {
  // 存储所有工具的完整元数据
  private static readonly tools: BaseToolMeta[] = [];

  /**
   * 注册工具（应用启动时调用）
   * @param toolMeta 工具完整元数据
   */
  static registerTool(toolMeta: BaseToolMeta): void {
    if (!toolMeta.toolName) {
      throw new Error("工具必须包含toolName字段");
    }
    this.tools.push(toolMeta);
  }

  /**
   * 获取所有工具的抽象元数据（仅意图分解需要的信息）
   * @returns 工具抽象元数据列表
   */
  static getToolCapabilities(): ToolCapabilityMeta[] {
    return this.tools.map(tool => ({
      capabilityType: tool.capabilityType,
      layer: tool.layer,
      sceneTags: tool.sceneTags,
      isAvailable: tool.isAvailable
    }));
  }

  /**
   * 获取所有工具的完整元数据（工具选择需要的信息）
   * @returns 工具完整元数据列表
   */
  static getAllTools(): BaseToolMeta[] {
    return [...this.tools]; // 返回副本，避免外部修改
  }

  /**
   * 根据能力类型筛选工具
   * @param capabilityType 工具能力类型
   * @returns 匹配的工具列表
   */
  static getToolsByCapability(capabilityType: ToolCapabilityType): BaseToolMeta[] {
    return this.tools.filter(tool => 
      tool.capabilityType === capabilityType && tool.isAvailable
    );
  }
}
```

### 5.1 历史会话处理器（intentDecomposition/utils/historyProcessor.ts）
```typescript
/**
 * 历史会话处理工具
 * 过滤无关内容、限制轮次，保证输入精简
 */
import { HistoryMessage } from "../../../common/types";

export class HistoryProcessor {
  // 默认最大保留轮次
  private readonly maxHistoryRounds: number;

  constructor(maxHistoryRounds: number = 8) {
    this.maxHistoryRounds = maxHistoryRounds;
  }

  /**
   * 处理历史会话
   * @param historyMessages 原始历史会话
   * @returns 过滤后的历史会话文本（用于Prompt）
   */
  process(historyMessages: HistoryMessage[]): string {
    // 1. 限制轮次（取最后N轮）
    const limitedHistory = historyMessages.slice(-this.maxHistoryRounds);
    
    // 2. 过滤无关内容（短文本、纯寒暄）
    const filteredHistory = limitedHistory.filter(msg => {
      const content = msg.content.trim();
      // 过滤空内容或长度<3的纯寒暄
      if (content.length < 3) return false;
      // 可扩展更多过滤规则（如关键词过滤）
      const ignoreKeywords = ["你好", "谢谢", "不客气", "好的"];
      return !ignoreKeywords.includes(content);
    });

    // 3. 转换为文本格式（便于Prompt输入）
    return filteredHistory.map(msg => `${msg.role}: ${msg.content}`).join("\n");
  }
}
```

### 5.2 分解Prompt模板（intentDecomposition/config/promptTemplates.ts）

```typescript
/**
 * 意图分解Prompt模板
 * 强制LLM输出结构化JSON，确保解析稳定性
 */
import { ToolCapabilityType } from "../../../common/types";

interface DecompositionPromptParams {
  coreIntent: string;
  history: string;
  availableCapabilities: Array<{

    sceneTags: string[];
  }>;

/**
 * 生成意图分解Prompt
 * @param params 模板参数
 * @returns 完整Prompt文本
 */
export function getDecompositionPrompt(params: DecompositionPromptParams): string {
  const { coreIntent, history, availableCapabilities } = params;
  
  // 格式化可用能力为文本
  const capabilitiesText = JSON.stringify(availableCapabilities, null, 2);

  return `
你是智能体的意图分解专家，需要将用户的核心意图拆分为可执行的原子步骤。

## 输入信息
1. 历史会话（已过滤无关内容）：
${history || "无"}

2. 用户核心意图：
${coreIntent}

3. 可用工具能力（仅能基于这些能力拆分步骤）：
${capabilitiesText}

## 输出要求
1. 必须输出JSON格式，字段如下：
   - steps: 步骤数组，每个步骤包含：
     - stepId: 数字，从1开始递增
     - stepContent: 字符串，步骤描述（简洁、可执行）
     - requiredCapability: 字符串，步骤所需工具能力（必须是可用能力中的类型）
     - layerPriority: 字符串，优先选择的工具分层（basic/advanced/expert）
     - dependencies: 数字数组，依赖的步骤ID（无依赖则为空数组）
     - backupStep: 字符串，可选，步骤失败时的兜底方案
2. 步骤必须满足：
   - 原子化：每个步骤仅对应一种工具能力
   - 有序性：按执行顺序排列，依赖步骤必须在前
   - 可行性：所有步骤的requiredCapability必须在可用能力列表中
   - 最少化：避免重复步骤，步骤总数不超过8个

## 错误禁止
- 禁止输出JSON以外的任何内容
- 禁止拆分出无对应工具能力的步骤
- 禁止忽略历史会话上下文
  `.trim();
}
```

### 5.3 意图分解核心类（intentDecomposition/core/decomposer.ts）
```typescript
/**
 * 意图分解核心逻辑
 * 负责将核心意图拆分为结构化步骤
 */
import { z } from "zod"; // 需安装：npm install zod
import { 
  IntentRecognitionResult, 
  HistoryMessage, 
  DecomposedStep,
  ToolRegistry,
  ToolCapabilityType,
  ToolLayer
} from "../../../common/types";
import { HistoryProcessor } from "../utils/historyProcessor";
import { getDecompositionPrompt } from "../config/promptTemplates";

// 定义LLM输出的Zod校验schema
const StepSchema = z.object({
  stepId: z.number(),
  stepContent: z.string(),
  requiredCapability: z.nativeEnum(ToolCapabilityType),
  layerPriority: z.nativeEnum(ToolLayer),
  dependencies: z.array(z.number()),
  backupStep: z.string().optional()
});

const DecompositionOutputSchema = z.object({
  steps: z.array(StepSchema)
});

export class IntentDecomposer {
  private readonly historyProcessor: HistoryProcessor;
  private readonly llmClient: any; // 替换为你的LLM客户端（如OpenAI）

  constructor(llmClient: any, maxHistoryRounds: number = 8) {
    this.historyProcessor = new HistoryProcessor(maxHistoryRounds);
    this.llmClient = llmClient; // 注入LLM客户端，解耦实现
  }

  /**
   * 核心分解方法
   * @param intentResult 意图识别结果
   * @param historyMessages 历史会话
   * @returns 结构化分解步骤
   */
  async decompose(
    intentResult: IntentRecognitionResult,
    historyMessages: HistoryMessage[]
  ): Promise<DecomposedStep[]> {
    try {
      // 1. 预处理历史会话
      const processedHistory = this.historyProcessor.process(historyMessages);

      // 2. 获取可用工具能力（仅抽象信息）
      const toolCapabilities = ToolRegistry.getToolCapabilities();
      const availableCapabilities = toolCapabilities.map(cap => ({
        type: cap.capabilityType,
        sceneTags: cap.sceneTags
      }));

      // 3. 构造Prompt
      const prompt = getDecompositionPrompt({
        coreIntent: intentResult.coreIntent,
        history: processedHistory,
        availableCapabilities
      });

      // 4. 调用LLM获取分解结果
      const llmResponse = await this.llmClient.chat.completions.create({
        model: "gpt-3.5-turbo", // 替换为你的模型
        messages: [{ role: "user", content: prompt }],
        temperature: 0.1, // 低随机性，保证结果稳定
        response_format: { type: "json_object" }
      });

      // 5. 解析并校验结果
      const responseText = llmResponse.choices[0].message.content || "{}";
      const parsedOutput = JSON.parse(responseText);
      const validatedOutput = DecompositionOutputSchema.parse(parsedOutput);

      // 6. 步骤优化（去重、排序、补全）
      const optimizedSteps = this.optimizeSteps(validatedOutput.steps);

      return optimizedSteps;
    } catch (error) {
      console.error("意图分解失败：", error);
      // 兜底：返回基础步骤（仅适配简单意图）
      return this.getFallbackSteps(intentResult.coreIntent);
    }
  }

  /**
   * 优化分解步骤
   * @param steps 原始步骤
   * @returns 优化后的步骤
   */
  private optimizeSteps(steps: DecomposedStep[]): DecomposedStep[] {
    // 1. 去重（基于stepContent和requiredCapability）
    const uniqueSteps = steps.filter((step, index, self) => 
      index === self.findIndex(s => 
        s.stepContent === step.stepContent && 
        s.requiredCapability === step.requiredCapability
      )
    );

    // 2. 按stepId排序
    const sortedSteps = uniqueSteps.sort((a, b) => a.stepId - b.stepId);

    // 3. 补全依赖（确保依赖步骤存在）
    return sortedSteps.map(step => ({
      ...step,
      dependencies: step.dependencies.filter(depId => 
        sortedSteps.some(s => s.stepId === depId)
      )
    }));
  }

  /**
   * 获取兜底步骤（分解失败时使用）
   * @param coreIntent 核心意图
   * @returns 基础步骤
   */
  private getFallbackSteps(coreIntent: string): DecomposedStep[] {
    return [
      {
        stepId: 1,
        stepContent: `处理用户意图：${coreIntent}`,
        requiredCapability: ToolCapabilityType.DATA_STATISTIC,
        layerPriority: ToolLayer.BASIC,
        dependencies: [],
        backupStep: "使用基础工具重试处理意图"
      }
    ];
  }
}
```

### 6.1 匹配规则配置（toolSelection/config/matchRules.ts）
```typescript
/**
 * 工具匹配规则配置
 */
import { ToolLayer, BaseToolMeta, DecomposedStep } from "../../common/types";

/**
 * 计算工具与步骤的匹配度（0-1）
 * @param tool 工具元数据
 * @param step 分解步骤
 * @returns 匹配度分数
 */
export function calculateMatchScore(tool: BaseToolMeta, step: DecomposedStep): number {
  let score = 0;

  // 1. 能力类型匹配（基础分）
  if (tool.capabilityType === step.requiredCapability) {
    score += 0.5;
  }

  // 2. 场景标签匹配（加分）
  const stepKeywords = step.stepContent.split(/\s+|，|。|、/).filter(kw => kw.length > 1);
  const matchedTags = tool.sceneTags.filter(tag => 
    stepKeywords.some(kw => tag.includes(kw) || kw.includes(tag))
  );
  score += (matchedTags.length / tool.sceneTags.length) * 0.3;

  // 3. 分层优先级匹配（加分）
  if (tool.layer === step.layerPriority) {
    score += 0.2;
  }

  return Math.min(score, 1); // 确保分数不超过1
}

/**
 * 工具排序规则（基础层优先，匹配度次之）
 * @param toolA 工具A
 * @param toolB 工具B
 * @param step 分解步骤
 * @returns 排序值
 */
export function sortToolsByPriority(
  toolA: BaseToolMeta,
  toolB: BaseToolMeta,
  step: DecomposedStep
): number {
  // 1. 分层优先级：基础层 > 进阶层 > 专家层
  const layerPriority: Record<ToolLayer, number> = {
    [ToolLayer.BASIC]: 3,
    [ToolLayer.ADVANCED]: 2,
    [ToolLayer.EXPERT]: 1
  };

  if (layerPriority[toolA.layer] !== layerPriority[toolB.layer]) {
    return layerPriority[toolB.layer] - layerPriority[toolA.layer];
  }

  // 2. 匹配度优先级
  const scoreA = calculateMatchScore(toolA, step);
  const scoreB = calculateMatchScore(toolB, step);
  return scoreB - scoreA;
}

/**
 * 生成工具调用参数
 * @param step 分解步骤
 * @param tool 选中的工具
 * @returns 工具参数
 */
export function generateToolParams(step: DecomposedStep, tool: BaseToolMeta): Record<string, any> {
  // 基础参数：从步骤中提取关键词
  const baseParams = {
    stepContent: step.stepContent,
    intentContext: step.stepContent
  };

  // 合并工具默认参数
  return {
    ...tool.defaultParams,
    ...baseParams
  };
}
```

### 6.2 工具选择核心类（toolSelection/core/toolMatcher.ts）
```typescript
/**
 * 工具匹配核心逻辑
 * 负责将分解步骤匹配到具体工具
 */
import { 
  DecomposedStep, 
  ToolSelectionResult, 
  ToolRegistry,
  BaseToolMeta
} from "../../common/types";
import { 
  calculateMatchScore, 
  sortToolsByPriority, 
  generateToolParams 
} from "../config/matchRules";

export class ToolMatcher {
  /**
   * 为分解步骤匹配具体工具
   * @param steps 分解后的步骤列表
   * @returns 工具选择结果列表
   */
  matchSteps(steps: DecomposedStep[]): ToolSelectionResult[] {
    return steps.map(step => {
      // 1. 获取匹配当前步骤能力的所有可用工具
      const candidateTools = ToolRegistry.getToolsByCapability(step.requiredCapability);

      if (candidateTools.length === 0) {
        // 无匹配工具，返回兜底结果
        return this.getFallbackSelectionResult(step);
      }

      // 2. 按优先级排序工具（基础层优先+匹配度）
      const sortedTools = candidateTools.sort((a, b) => 
        sortToolsByPriority(a, b, step)
      );

      // 3. 选择最优工具（第一个）
      const bestTool = sortedTools[0];

      // 4. 计算匹配度
      const matchScore = calculateMatchScore(bestTool, step);

      // 5. 生成工具参数
      const toolParams = generateToolParams(step, bestTool);

      // 6. 构造选择结果
      return {
        stepId: step.stepId,
        selectedTool: bestTool.toolName,
        toolLayer: bestTool.layer,
        toolParams,
        stepContent: step.stepContent,
        matchScore,
        backupTool: sortedTools[1]?.toolName // 第二个工具作为兜底
      };
    });
  }

  /**
   * 获取兜底工具选择结果（无匹配工具时）
   * @param step 分解步骤
   * @returns 兜底结果
   */
  private getFallbackSelectionResult(step: DecomposedStep): ToolSelectionResult {
    return {
      stepId: step.stepId,
      selectedTool: "default_tool",
      toolLayer: ToolLayer.BASIC,
      toolParams: {
```

```typescript
 * 为分解步骤匹配具体工具
 * @param steps 分解后的步骤列表
```
